Importando um arquivo javascript

Para utilizar javascript no seu HTML, você pode tanto importar um javascript externo com a tag <script> e passando o atributo src e passando como valor o caminho até o seu arquivo.

Ex: <script src="./script" ></script>

Ou você pode simplesmente escrever o seu script dentro da tag <script>

Comentários

O javascript possui duas formas de se escrever comentários. Ok, mas o que são comentários? São textos que apesar de estar dentro do seu código, o compilador ou navegador não irá executar aquele código. Temos comentários de uma linha só, e comentários de mais de uma linha. As duas formas são:

  • 1 Linha: (duas barras)
    // seu comentário vai aqui
  • Múltiplas linhas: (barra asterisco asterisco barra)
    /* Seu comentário vai aqui
    e aqui
    e aqui também. tudo o que estiver
    dentro das barras com asteriscos não será interpretado. */

Variáveis e constantes (let, const, var)

Variáveis são como espaços na memória do computador em que armazenamos informações por tempo de execução. Ok, mas o que isso significa? Significa que durante a execução do seu código, um determinado valor será guardado dentro de uma variável, que você pode entender como uma gaveta onde vc guarda uma o que vc quiser. Porém, esses valores guardados podem ser fixos ou sofrer alterações. Mas não se preocupe, porque isso está diretamente ligado à diferença entre as formas de se declarar uma variável.

Para declarar uma variável, nós utilizamos uma palavra-reservada. E o que são palavras-reservadas? São palavras reservadas pelo próprio javascript e que elas possuem um significado especifico para a linguagem. Exemplos de palavras reservadas "function", "import", "break", "continue", etc.

As palavras-reservadas para se declarar uma variável são:

  • let

    Ao declarar uma variável com let, você está querendo dizer que você está passando um valor para ela, porém esse valor pode ser alterado no futuro.

  • const

    Como o próprio nome sugere, ao se criar uma variável com const você está querendo dizer que o valor que vc irá atribuir à ela não será alterado de forma alguma

  • var

    Criar uma variável com var não é muito indicado, mas você pode. Mas por que não se deve criar uma variável com "var"?

    Porque ele transforma aquela variável que vc criou, em uma variável global. Ou seja, você pode acessar essa mesma variável em qualquer parte do seu projeto. Mas isso parece bom, não é? Mas não é, você pode terminar atribuindo um novo valor pra ela sem perceber, já que uma variável criada com "var" pode ser alterada, assim como uma variável com "let".

Na dúvida, usa let ou const. Quer criar uma variável que será mutável? usa LET. Quer criar uma variável que não será alterada? usa CONST. Entendeu?

Mostrando mensagens em tela (e no terminal)

Com javascript, nós temos diversas formas de interagir com o usuário e não só enviar informações para eles, como também receber informações do usuário.

Para enviar mensagens para o usuário, temos algumas formas e cada uma delas possuem um propósito. Vamos aos comandos:

  • alert()

    O "alert" serve para enviar ao usuário alguma mensagem relevante ou apenas um alerta mesmo.

  • prompt()

    O "prompt" ele abre uma caixa de texto no topo da tela assim como no "alert", porém, ele também abre uma caixa de texto para que o usuário insira alguma informação, e nós podemos pegar essa informação e manipulá-la.

  • confirm()

    O "confirm" também interage com o usuário, mas ele abre uma caixa de texto com duas opções "OK" ou "Cancelar", que podemos entender como "true" e "false". Com base nesse valores de verdadeiro e falso, podemos apresentar conteúdos diferentes.

  • console.log()

    É um método para mostrar uma mensagem ou qualquer outro tipo de valor no terminal, seja ele do navegador ou do seu próprio prompt de comando. Esse é a única forma de se enviar mensagens em tela que funciona tanto no navegador, quanto no sistema operacional (mas vc precisa ter o node instalado no pc).
